# Задание 1. Анализ и планирование #
### 1. Описание функциональности монолитного приложения ###
**Управление отоплением:**
- Пользователи могут:
  - Пользователи могут выполнять CRUD-операции с датчиками через API
  - Пользователи могут удалённо включать/выключать отопление в своих домах.
- Система поддерживает
  - Интеграцию с внешним temperature-api для получения значений данных

**Мониторинг температуры:**
- Пользователи могут:
    - просматривать текущую температуру в своих домах через веб-интерфейс.
- Система поддерживает:
    - получение данных о температуре/статусе/местонахождении с датчиков, установленных в домах.

### 2. Анализ архитектуры монолитного приложения
- Язык программирования: Go
- База данных: PostgreSQL
- Архитектура: монолитная архитектура, имеется одно приложение, обрабатывающее запросы, работающее с данными,
- Взаимодействие: синхронное, через HTTP-запросы
- Масштабирование: используется монолитное приложение, масштабирование возможно только для всего монолита
- Развертывание: требует остановки всего приложения, подготовлен docker-compose и init.sql файлы для запуска приложения

### 3. Определение доменов и границы контекстов
Домены:
- Управление пользователями (User service)
- Управление устройствами (Device service)
  - Поддомен управления домами
  - Поддомен управления модулями
  - Поддомен управления датчиками
- Телеметрия и аналитика (Telemetry and analytics service)
  - Поддомен: сбор данных с датчиков
  - Поддомен: сбор аналитики
- Оповещения (Notification service)
- Управление платежами (Payment service)

### **4. Проблемы монолитного решения**
- Масштабируемость: трудности в масштабировании при росте нагрузки на отдельные компоненты системы
- Единая точка отказа: при отказе монолита, весь функционал недоступен пользователям
- Ограничения в технологическом стеке: в монолите используется Go, нет возможности использовать более подходящие инструменты под конкретные заадчи
- Ограничения в функциональности: Поддерживается только температурные датчики и вкл/выкл отопления, установка новых датчиков только через выезд специалиста
- Синхронное взаимодействие: сбор данных со всех датчиков при синхронном взаимодействии может повлиять на скорость работы системы
- Развертывание: при внесении изменений в одной части системы, необходимо развертывать всю систему целиком
